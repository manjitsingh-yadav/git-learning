6. Reset - Moves HEAD pointer to a specific commit and it may also change some of the files in the staging index or working directory.
	6.1 Soft - Just moves the HEAD pointer and does not change staging index and working directory.
		     Why? Return to an old state and leave code changes staged. Useful for amending one or more commits.
		     - git reset --soft <treeish>
	6.2 Mixed - Moves the HEAD pointer and changes the staging index to match repository. This is the defualt choice if no type is mentioned.
		     Why? Return to an old state and leave code changes in working directory. Useful for reorganizing commits.
	6.3 Hard - Moves HEAD pointer, changes staging index to match the repo and changes the working directory to match repository.
		     Why? Return to an old state and discard all subsequent code changes. Useful to permanently undo commits.

8. git stash save "changed mission page title" | git stash -u save (to include untracked files as well)
   git checkout master
   git stash list
   git stash show stash@{0} | git stash show -p stash@{0}
   git stash pop (will apply and delete the stash)
   git stash apply (will apply the stash but won't delete it)
   git stash drop stash@{0}
   git stash clear (clears everything)
9. Set up Remote: git remote | git remote -v 
			git remote add origin URL
			cat .git/config
   			git push -u origin master
			git remote rm origin
10. Clone a repo: git clone repo-URL <directory-optional> (Cloning only brings down the master branch)
11. Track remote branch: git branch -u origin/non-tracking non-tracking | git push -u origin master (post this you can straight away issue "git push" without mentioning origin and the branch name)
				 git log --oneline -5 origin/non-tracking | git log --oneline -5 origin/master
12. git pull (= git fetch + git merge origin/master)
13. Checkout remote branches: git pull (this pulls all new codes and remote branches)
					git branch non-tracking origin/non-tracking (this is to work on a branch that was pushed to remote repository by someone else and now you want to have it locally)
						OR
					git checkout -b non-tracking origin/non-tracking
13. Delete remote branch: git branch -r
				 git push origin :non-tracking | git push origin --delete non-tracking
14. Enable collaboration: a. Private repos: Sign in to git hub -> go to repo -> settings tab -> Collaborators
				  b. Public repos: Check Issues and Pull Requests tab first -> Fork the repo -> Complete your code -> Create Pull Request
